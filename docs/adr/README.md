# Architecture Decision Records (ADRs)

This directory contains Architecture Decision Records for the CAF (Clean Architecture Frontend) project. ADRs document significant architectural decisions, their context, and consequences.

|-----|--------|
| [ADR-001](001-caf-folder-structure.md) | The `caf/` folder structure |
| [ADR-002](002-pulse-vs-ploc.md) | Pulse vs Ploc: when to use each |
| [ADR-003](003-routing-abstraction.md) | Routing abstraction (RouteManager / RouteRepository) |
| [ADR-004](004-package-separation.md) | Package separation (monorepo and scoped packages) |

## Format

Each ADR follows a simple structure:

- **Title** — Short descriptive name
- **Status** — Proposed | Accepted | Deprecated | Superseded
- **Context** — What is the issue we are facing?
- **Decision** — What did we decide?
- **Consequences** — What are the results (positive and negative)?
